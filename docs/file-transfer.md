# Foundation Sunshine 文件映射与高级全盘文件管理

本 fork 默认复用上游 File Mapping 地基：授权 mapping、会话 token、独立 WSS、路径校验和配置事务都沿用上游实现，只提供授权目录的只读访问。BuffPlum 仅在显式开启的实验模式中增加全盘枚举、写操作、冲突策略、双栏 UI 与串流拖放。

## 为什么做这个功能

这个功能来自一个很实际的局域网使用场景：两台自己的电脑就在同一个可信网络中，临时传一个文件却要来回拿 U 盘，或者先发到 QQ、微信再下载。最初开始实验时，Sunshine 已经可以用很高的画质和较低的延迟操控另一台电脑，却还没有与这种远程操作体验配套的文件传输；现在上游已有只读 File Mapping，本 fork 因而收缩为只维护高级全盘读写差异。

上游现在已经维护安全、通用的只读目录共享底座，这是本 fork 的默认路径。上游没有维护的是全盘读写、覆盖和删除这一组高风险扩展；一旦配对设备、账号或局域网环境不可信，影响范围可能覆盖 Sunshine 进程有权访问的全部文件。因此 BuffPlum 只维护这层差异，不复制上游公共地基，也不代表上游立场。

## 两种模式

### 上游兼容只读模式（默认）

```ini
file_mapping_mode = read_only
```

- 只公开通过 Control Panel、本机管理 API 或配置文件明确授权的目录。
- 配置中的 `readwrite` 和 `allow_delete` 会被降级并记录警告。
- Moonlight 显示上游兼容的 **Host Files** 只读入口。
- 不启用双栏写入、新建、重命名、删除或串流窗口拖放上传。

### BuffPlum 全盘读写模式（实验性）

```ini
file_mapping_mode = full_disk
```

- 动态枚举 Sunshine 进程当前可以访问的磁盘，不需要逐个添加共享目录。
- capability 明确返回 `access_mode: full_disk` 与 `buffplum_full_disk`，Moonlight 只有收到这些能力后才启用高级 UI。
- 支持上传、下载、新建、重命名、删除、冲突策略与窗口化串流拖放。
- 通过管理 API 修改模式后，应断开并重新建立串流，使客户端重新获取 capability 和 mapping 列表；直接编辑配置文件则需要重启 Sunshine。

管理 API 也提供模式读取与更新：

```text
GET   /api/v1/file-mapping/mode
PATCH /api/v1/file-mapping/mode
      {"mode":"read_only"}
      {"mode":"full_disk"}
```

模式写入使用与 mapping 管理相同的事务锁；持久化失败时不会改变运行时模式。

> 从旧版 BuffPlum 升级后，默认会回到 `read_only`。确实需要原有全盘能力的用户必须显式设置 `full_disk`。

## 全盘模式使用方法

1. 打开 Sunshine Web UI 的 **配置 → 文件传输 → 文件访问模式**，选择 **BuffPlum 全盘读写模式（实验性）**，阅读风险提示、勾选确认并再次点击启用。该操作通过本机管理 API 立即持久化，不需要点击页面右下角的普通保存按钮；也可以手动设置 `file_mapping_mode = full_disk` 后重启 Sunshine。
2. 使用配套 Moonlight 完成配对并启动桌面串流。
3. 按 `Ctrl+Alt+Shift+O` 打开悬浮菜单，选择 **Host Files / 主机文件**。
4. 左侧选择客户端磁盘和目录，右侧选择主机的 `C:`、`D:`、`E:` 等磁盘和目录。
5. 选中文件或文件夹后点击 **Upload →** 或 **← Download**。
6. 也可以在窗口化串流时，把控制端资源管理器里的文件或文件夹直接拖进串流窗口；Moonlight 会发送到设置中指定的主机接收目录。

磁盘列表在建立文件传输连接时动态生成，因此新插入并可访问的移动磁盘可以在重新打开文件传输窗口后出现。空光驱和断开的网络磁盘不会显示；光盘只提供下载。

## 全盘模式传输行为

- 文件和整个目录都可以双向传输。
- Moonlight 在后台线程中按 256 KiB 分块读取或写入，大文件传输不会阻塞串流事件循环。
- Sunshine 先写 `.moonlight-upload-*.part` 临时文件，最后一个分块完成后才原子提交。
- 下载也先写入客户端临时文件，完整接收后再改成最终文件名。
- 默认使用 **保留两者**：同名文件或文件夹会按 Windows 资源管理器习惯保存为 `名称 (1)`、`名称 (2)` 等。
- 用户可以明确切换为 **覆盖**。文件仍通过临时文件完成后再替换，避免留下半个文件；覆盖已有目录时会合并其内容。
- 文件传输窗口支持新建文件、新建文件夹、重命名和删除。删除前必须再次确认；磁盘根目录和上传临时文件始终禁止删除。
- Moonlight 设置中可以填写主机接收目录，也可以在文件传输窗口进入某个远程目录后点击 **设为接收目录**。
- 不提供远程执行功能。

## 权限与安全

- 默认 `read_only` 模式不会枚举全盘，也不会开放写入。
- HTTPS 能力接口只接受已经配对的客户端；数据 WebSocket 还要求绑定客户端 UUID 的一次性短期令牌。
- 每个磁盘是独立根节点，例如 `drive-c` 对应 `C:\`。协议中的路径仍必须是该磁盘下的相对路径。
- 绝对路径、`..`、Windows 保留名、符号链接和目录联接穿越都会被拒绝。
- 可见范围取决于 Sunshine 进程的 Windows 账号权限。受 ACL 保护的系统目录不会因为文件传输功能而绕过权限。
- Sunshine 作为 Windows 服务运行时，映射到交互用户的网络盘可能对服务账号不可见。
- 配对认证并不能消除误操作、恶意客户端、会话令牌泄露或主机账号权限过高带来的风险。只应在自己的设备和可信局域网中启用，并提前备份重要数据。
